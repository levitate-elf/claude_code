# 花卉异常分类实验说明

## 1. 任务定义

- 目标：基于 `5-fold` 中的训练/验证/测试划分，训练花卉异常分类模型，并在测试集上报告 `Accuracy` 和 `F1`。
- 标签来源：`瑕疵表_index.csv`
- 标签去重后理论类别数：`87`

## 2. 当前数据的实际情况

- `5-fold` 这批样本里，实际有图像样本的去重异常类别只有 `38` 类。
- 其余 `49` 类只在 `瑕疵表_index.csv` 里出现，但在 `5-fold` 训练数据中没有任何样本，因此当前数据无法真正训练出完整的 `87` 类闭集分类器。
- 数据清洗时做了以下处理：
  - 过滤缺失图片
  - 删除同一图片对应多个冲突标签的样本
  - 删除 train / val / test 间重复图片，避免信息泄漏
  - 对同一 split 内重复图片去重

## 3. 方案

### 3.1 传统基线

- 脚本：`train_flower_defect_baseline.py`
- 特征：`HOG + HSV 颜色统计 + LBP纹理`
- 模型：Scikit-learn 线性分类器

5-fold 平均测试结果：

- Accuracy: `0.2636`
- Macro F1: `0.0755`
- Weighted F1: `0.2554`

### 3.2 图像模型

- 脚本：`train_flower_defect_torch.py`
- 环境：`E:\anaconda3\envs\py312\python.exe`
- 模型：`ResNet18(ImageNet 预训练) + 全连接分类头`
- 训练策略：
  - 先冻结 backbone 训练 1 个 epoch
  - 再全量 fine-tune 3 个 epoch
  - 使用随机裁剪、翻转、颜色扰动增强
  - 使用加权采样缓解类别不均衡

## 4. 最终 5-fold 测试结果

平均结果：

- Accuracy: `0.3577`
- Macro F1: `0.1524`
- Weighted F1: `0.3462`

各 fold 测试结果：

- `fold_1`: Accuracy `0.3724`, Macro F1 `0.1671`
- `fold_2`: Accuracy `0.3699`, Macro F1 `0.1588`
- `fold_3`: Accuracy `0.3319`, Macro F1 `0.1243`
- `fold_4`: Accuracy `0.3445`, Macro F1 `0.1522`
- `fold_5`: Accuracy `0.3699`, Macro F1 `0.1598`

### 3.3 结构化特征模型

- 脚本：`train_flower_defect_metadata.py`
- 模型：`One-Hot + StandardScaler + LogisticRegression`
- 使用字段：
  - 日期
  - 供货代码 / 供货名称
  - 品类代码 / 品类名称
  - 品种代码 / 品种名称
  - 等级
  - 数量
  - 长度
  - 成熟度
  - 底价

严格 `train.csv -> test.csv` 口径的 5-fold 平均结果：

- Accuracy: `0.4411`
- Macro F1: `0.2569`
- Weighted F1: `0.4558`

如果在模型选型完成后，将 `val.csv` 也并入最终训练集，再对 `test.csv` 评估：

- Accuracy: `0.4489`
- Macro F1: `0.2712`
- Weighted F1: `0.4676`

### 3.4 图像 + 表格融合模型

- 脚本：`train_flower_defect_fusion.py`
- 融合输入：
  - 表格模型：`train_flower_defect_metadata.py`
  - 图像模型：`train_flower_defect_torch.py`
- 融合方式：对两边输出的类别概率做后融合

#### v1: 加权平均融合

- 目录：`artifacts_fusion_v1_additive_valacc`
- 权重选择：在各 fold 的 `val.csv` 上按 `Accuracy` 选 alpha
- 5-fold 平均：
  - Accuracy: `0.4688`
  - Macro F1: `0.2708`
  - Weighted F1: `0.4717`

#### v2: 几何融合

- 目录：`artifacts_fusion_v2_geometric_valacc`
- 权重选择：在各 fold 的 `val.csv` 上按 `Accuracy` 选 alpha
- 5-fold 平均：
  - Accuracy: `0.4933`
  - Macro F1: `0.2860`
  - Weighted F1: `0.4914`

#### v3: 最终重训版

- 目录：`artifacts_fusion_v3_geometric_refit_meta_trainval`
- 做法：
  - 先用 `v2` 的验证集结果确定每个 fold 的融合权重
  - 再把表格模型用 `train + val` 重训
  - 图像模型沿用已有版本
- 5-fold 平均：
  - Accuracy: `0.5046`
  - Macro F1: `0.3082`
  - Weighted F1: `0.5037`

## 7. 下一步建议

- 如果你希望真正做 `87` 类分类，必须先补齐那 `49` 个缺失类别的训练样本。
- 如果后续允许继续提升，可以再做：
  - 更强的 backbone，例如 `EfficientNet / ConvNeXt / ViT`
  - 类别重采样和 focal loss
  - 图像和结构化字段做多模态融合
  - 分层任务设计：先做大类异常检测，再做细粒度异常分类
