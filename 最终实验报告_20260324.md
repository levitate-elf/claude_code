# 花卉异常分类任务实验报告

## 1. 任务说明

- 目标：基于 `5-fold` 数据，对花卉异常类别进行分类，并在测试集上报告 `Accuracy` 和 `Macro F1`。
- 标签来源：`瑕疵表_index.csv`
- 理论异常类别数：`87`

## 2. 数据说明

### 2.1 实际可训练类别

- `瑕疵表_index.csv` 去重后共有 `87` 类异常。
- 但当前 `5-fold` 实际有样本的类别只有 `38` 类。
- 另外 `49` 类在当前训练数据中没有样本，因此本实验实际完成的是 `38` 类异常分类。

### 2.2 数据清洗

所有实验都统一做了以下清洗：

- 过滤缺失图片样本
- 去除同一图片对应多个冲突标签的样本
- 去除 train / val / test 之间重复图片，避免数据泄漏
- 去除同一 split 内的重复样本

## 3. 方法演化

### 3.1 传统图像基线

- 方法：`HOG + HSV颜色统计 + LBP + 线性分类器`
- 脚本：`train_flower_defect_baseline.py`

结果：

- 5-fold 平均 Accuracy：`0.2636`
- 5-fold 平均 Macro F1：`0.0755`

### 3.2 纯图像深度学习模型

- 方法：`ResNet18(ImageNet预训练) + 微调`
- 脚本：`train_flower_defect_torch.py`

结果：

- 5-fold 平均 Accuracy：`0.3577`
- 5-fold 平均 Macro F1：`0.1524`

### 3.3 结构化特征模型

- 方法：`One-HotEncoder + StandardScaler + LogisticRegression`
- 使用字段：
  - 日期
  - 供货代码 / 供货名称
  - 品类代码 / 品类名称
  - 品种代码 / 品种名称
  - 等级
  - 数量
  - 长度
  - 成熟度
  - 底价
- 脚本：`train_flower_defect_metadata.py`

结果：

- `train -> test`：Accuracy `0.4411`，Macro F1 `0.2569`
- `train + val -> test`：Accuracy `0.4489`，Macro F1 `0.2712`

### 3.4 图像 + 表格融合模型

- 方法：对图像模型和表格模型输出的类别概率进行后融合
- 脚本：`train_flower_defect_fusion.py`

实验版本：

| 版本 | 说明 | 平均 Accuracy | 平均 Macro F1 |
|---|---|---:|---:|
| `fusion_v1` | 加权平均融合，按验证集 Accuracy 选权重 | `0.4688` | `0.2708` |
| `fusion_v2` | 几何融合，按验证集 Accuracy 选权重 | `0.4933` | `0.2860` |
| `fusion_v3` | 在 `fusion_v2` 权重基础上，用 `train + val` 重训表格模型后再融合 | `0.5046` | `0.3082` |

## 4. 最终采用方案

最终最佳方案为：`fusion_v3`

方案组成：

- 图像分支：`ResNet18` 微调模型
- 表格分支：`LogisticRegression` 结构化特征模型
- 融合方式：`geometric fusion`
- 融合权重：沿用 `fusion_v2` 在各 fold 验证集上选出的最优权重

最终 5-fold 平均结果：

- Accuracy：`0.5046`
- Macro F1：`0.3082`
- Weighted F1：`0.5037`

## 6. 各版本结果总览

| 方法 | 训练口径 | 平均 Accuracy | 平均 Macro F1 |
|---|---|---:|---:|
| 传统图像基线 | `train -> test` | `0.2636` | `0.0755` |
| 纯图像 ResNet18 | `train -> test` | `0.3577` | `0.1524` |
| 表格模型 | `train -> test` | `0.4411` | `0.2569` |
| 表格模型 | `train + val -> test` | `0.4489` | `0.2712` |
| 融合 `v1` | `train -> val 调权重 -> test` | `0.4688` | `0.2708` |
| 融合 `v2` | `train -> val 调权重 -> test` | `0.4933` | `0.2860` |
| 融合 `v3` | `train + val 重训表格 + 融合` | `0.5046` | `0.3082` |

## 7. 结果解释

本任务中，结构化字段对异常类别具有较强判别力，因此表格模型本身已经显著优于纯图像模型。在此基础上，再将图像模型作为补充信息加入融合，能够进一步提升准确率和 F1。

最终最优方案不是单一 CNN，而是“图像模型 + 表格模型”的融合方案。

## 9. 注意事项

- 当前结果基于 `38` 个实际有样本的异常类别，不是完整 `87` 类闭集分类结果。
- 如果后续要做真正的 `87` 类分类，需要先补齐那 `49` 个缺失类别的训练样本。
- 如果部署场景中不允许使用表格字段，则应以纯图像模型结果为准，而不能直接使用当前融合模型的分数。
